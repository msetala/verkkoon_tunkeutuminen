- tiedoston ladattuani avasin sen komenolla `rtl_433 -r <tiedostonimi>`, ja ohjelma aukesi:

- <b>tunnistettavia laitteita (model):</b> `Nexa-Security`, `KlikAanKlikUIT-Switch`, `Proove-Security`
- <b>housecode</b> on jokaisella laitteella sama (8785315). netistä kaivetun tiedon mukaan tämä voi osoittaa laitteiden jakavan saman "omistajan", mutta halvoissa laitteissa se voi olla aina sama.

Veikkaisin valonhimmennintä tai pistorasiaa muiden tietojen perusteella (dim/off). <b>Proove-Security</b> Näyttäisi olevan samanlainen. KlikAanKlikUit on ainoa laite jossa komento on off, eli tämä on varsinainen kaukosäädin, joka antaa muille käskyjä.

### e) Ultimate. URH-asennus
- Urhin kerkesinkin jo asentaa tunnilla, mutta hyödynsin komentoja, mitä vinkit osiostakin löytyy:
```
$ sudo apt-get update
$ sudo apt-get -y install pipx
$ pipx install urh
$ pipx ensurepath
```
- ... näyttäisi kuitenkin siltä, että ohjelma on jotenkin hajonnut. vaikka urh toimikin tunnilla, nyt saan <b> illegal instruction </b> palautteen aina kun yritän avata urh:in.
- Tässä vaiheessa oli konsultoitava tekoälyä korjausmielessä, sillä nettifoorumeista ei löytynyt mitään ongelmaani. Tuli kokeiltua eri korjausmekanismeja pipx:n ja urhin uudelleenasetuksista virtualboxin asetuksien muokkaamiseen, mikään ei auttanut.
- Laitan tässä vaiheessa palautukseen, päivitän jos saan korjattua. Jos ei, joudun pyytämään maanantain tunnilta apua ja palautan tunnin jälkeen.

- <b> Päivitys 10.5:</b> sain ristiinarvioinnissa vinkin, että voisi kokeilla venv-ympäristöllä. Olin tätä jo kokoeillut aiemmin, turhauduin ja kysyin vielä tänään tekoälyltä.
