### x) vastaa lyhyesti kysymyksiin

- <b>tuskan pyramidi</b> on kyberturvallisuuden malli, joka luokittelee erityyppisiä uhkaindikaattoreita sen perusteella, kuinka vaikeaa hyökkääjien on muuttaa strategiaansa ne havaittuaan.
- <b>tunkeutumisanalysiin timanttimalli</b> on kyberturvallisuuden kehys, jonka tarkoitus on auttaa organisaatioita analysoimaan kyberhyökkäyksiä. Timanttimallin neljä peruskomponenttia ovat: hyökkääjä, infrastruktuuri(hyökkääjän resurssit), kyvyt(metodit, työkalut tai tekniikat) sekä uhri.



### a) apache log

- itse apachen asennuksesta minulta ei löydy dokumentointia, sillä se löytyy jo koneeltani. kuitenkin apachen testisivulle asti päästään komennoilla:
```
$ sudo apt install apache2
$ firefox "http://localhost"
```
- apachen lokien päätyyn päästään komennolla
```
$ sudo tail -F /var/log/apache2/access.log
```
<img width="700" height="49" alt="kuva" src="https://github.com/user-attachments/assets/09d9fa0b-c508-49f1-9d38-bb9ba3a0aa63" />
<br></br>

- `127.0.0.1` asiakkaan ip osoite (kuka teki pyynnön palvelimelle)
- `--` tarkoittaa että tietoa ei ole saatavilla. ensimmäisen viivan kuuluisi olla asiakkaan RFC 1413 identiteetti. toinen viiva olisi käyttäjän identiteetti (user id). koska dokumentti ei ole salattu sanasalla, uid ei näytetä.
- pyynnön aikaleima: päivä/kuukausi/vuosi/tunnit/minuutit/sekunnit/second zone(?)
- `GET / HTTP/1.1` metodi GET, `/` pyydettävä resurssi (tyhjä hakemisto), protokolla HTTP/1.1 tarkoittaa että yhteys jää auki. Eliminoi TCP-kättelyiden tarpeen jokaiselle pyynnölle.
- `200` on status jonka palvelin antaa vastauksena asiakkaalle. 2:lla alkuiset koodit tarkoittavat onnistumista, 3:lla alkavat uudelleenohjausta ja 4:lla alkavat virhetilaa.
- `3383` on objektin koko palvelimelta asiakkaalle
- loppuosa user-agentin pyyntöotsikko, eli tietoa jota selain antaa itsestään

### b) nmapped

- nmapin asennus hoituu komennolla `sudo apt install nmap`
- skannaus on tehty komennolla `nmap -A "localhost"`

- `PORT: 80/TCP` näyttää portin ja sen protokollan
- `STATE: open` viittaa aukenaiseen porttiin
- `SERVICE: http` viittaa http-palvelimeen
- `VERSION: apache httpd 2.4.66 ((Debian))` sivusto pyörii apachen versiolla 2.4.66, Debianilla. d viittaa demoniin.

- mennään uudestaan lokeihin samalla komennolla kuin aiemmin:

<img width="700" height="353" alt="kuva" src="https://github.com/user-attachments/assets/5ac191f8-5b5d-4d7b-9e78-493bcca79f93" />

- nmap löytyy monta kertaa katsoessa lokeja komennolla `sudo tail -F /var/log/apache2/access.log`.
- silmään iskee heti `OPTIONS`, http metodi joka pyytää sallittuja kommunikaatiovaihtoehtoja URL:ille tai palvelimelle.
- enemmän vastauksia sai pelkällä `cat` komennolla, kuvassa näkyvät `PROPFIND` JA `IMFT`. propfind näyttäisi kyselevän metadataa, kuten tiedoston kokoa, luontipäivää ja luojan tietoja XML-muodossa. seuraavasta metodista en löytänyt verkosta tietoa. 
- kuvassa näkyvä `GET evox/about` on oletusarvoinen `HTTP` skriptipyyntö, joka etsii tiettyjä haavoittuvuuksia tai tunnistaa ohjelmistoversioita
- voit suodattaa "nmap" koskevat lokit komennolla
```
 cat /var/log/apache2/access.log | grep "nmap"
```

### e) wire-sharking

- kuvan tilanteeseen päästään avaamalla komentokehotteelta kaksi ikkunaa, ensimmäinen käynnistää wiresharkin `sudo wireshark` komennolla.
- wireshark laitetaan kaappaamaan loopback:ia `lo`
- tämän jälkeen ajetaan toisessa komentokehotteen ikkunassa komento `nmap -A "localhost"`

<img width="700" height="330" alt="kuva" src="https://github.com/user-attachments/assets/a9c22e0d-6019-457c-b6a4-3f86e13a74f4" />
<br></br>

- yläosan suodattimiin lisätty `frame contains "nmap"`
- kaappaus näyttäisi vastaavan apachen lokeja, paljon GET, PROPFIND ja POST metodeja

### f) net-grep

- asensin ensin ngrepin (joka koneestani puuttui), komennolla `sudo apt install ngrep`.
- ensimmäisessä ikkunassa käynnistin ohjelman komennolla `sudo ngrep -d lo -i nmap`, ja toisessa käynnistin nmapin `nmap -A "localhost"`
- tulokset ovat seuraavanlaiset:

<img width="700" height="657" alt="kuva" src="https://github.com/user-attachments/assets/1194a75e-8666-4ee4-9e29-67925e69e684" />
<br></br>

- näiden kolmen viimeisen tehtävän perusteella tuntuu, että jokainen ohjelma antaa (melko) samanlaiset vastaukset, eli vaihtoehtoja skannausten tekemiseen on monta.
- POST, PROPFIND, GET ja POST löytyvät täältäkin.

### g) agentti & h) pienemmät jäljet

- vaihdoin user agentin opettajan vihjettä käyttäen antamalla komennon:
```
nmap -A localhost --script-args http.useragent="Mozilla/5.0 (X11; Linux x86_64; rv:140.0) Gecko/20100101 Firefox/140.0"
```
- eli tässä tapauksessa vinkistä `http.useragent="` lähtien XBox on korvattu Mozillan tunnisteella (ks. kuva tehtävässä apache-log).
