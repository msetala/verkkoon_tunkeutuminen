### a) apache log

<img width="1282" height="49" alt="kuva" src="https://github.com/user-attachments/assets/09d9fa0b-c508-49f1-9d38-bb9ba3a0aa63" />

- `127.0.0.1` asiakkaan ip osoite (kuka teki pyynnön palvelimelle)
- `--` tarkoittaa että tietoa ei ole saatavilla. ensimmäisen viivan kuuluisi olla asiakkaan RFC 1413 identiteetti. toinen viiva olisi käyttäjän identiteetti (user id). koska dokumentti ei ole salattu sanasalla, uid ei näytetä.
- pyynnön aikaleima: päivä/kuukausi/vuosi/tunnit/minuutit/sekunnit/second zone(?)
- `GET / HTTP/1.1` metodi GET, `/` pyydettävä resurssi (tyhjä hakemisto), protokolla HTTP/1.1 tarkoittaa että yhteys jää auki. Eliminoi TCP-kättelyiden tarpeen jokaiselle pyynnölle.
- `200` on status jonka palvelin antaa vastauksena asiakkaalle. 2:lla alkuiset koodit tarkoittavat onnistumista, 3:lla alkavat uudelleenohjausta ja 4:lla alkavat virhetilaa.
- `3383` on objektin koko palvelimelta asiakkaalle
- loppuosa user-agentin pyyntöotsikko, eli tietoa jota selain antaa itsestään

### d) jäljet lokeissa

<img width="1265" height="353" alt="kuva" src="https://github.com/user-attachments/assets/5ac191f8-5b5d-4d7b-9e78-493bcca79f93" />

<img width="1266" height="99" alt="kuva" src="https://github.com/user-attachments/assets/1c3c5545-ce95-413f-a341-9cfe82239e47" />


- nmap löytyy monta kertaa katsoessa lokeja komennolla `sudo tail -F /var/log/apache2/access.log`.
- silmään iskee heti `OPTIONS`, http metodi joka pyytää sallittuja kommunikaatiovaihtoehtoja URL:ille tai palvelimelle.
- enemmän vastauksia sai pelkällä `cat` komennolla, kuvassa näkyvät `PROPFIND` JA `IMFT`. propfind näyttäisi kyselevän metadataa, kuten tiedoston kokoa, luontipäivää ja luojan tietoja XML-muodossa. seuraavasta metodista en löytänyt verkosta tietoa. 
- kuvassa näkyvä `GET evox/about` on oletusarvoinen `HTTP` skriptipyyntö, joka etsii tiettyjä haavoittuvuuksia tai tunnistaa ohjelmistoversioita
- voit suodattaa "nmap" koskevat lokit komennolla
```
 cat /var/log/apache2/access.log | grep "nmap"
```

### f) net-grep

- asensin ensin ngrepin (joka koneestani puuttui), komennolla `sudo apt install ngrep`.
- ensimmäisessä ikkunassa käynnistin ohjelman komennolla `sudo ngrep -d lo -i nmap`, ja toisessa käynnistin nmapin `nmap -A "localhost"`
- tulokset ovat seuraavanlaiset:

<img width="1262" height="657" alt="kuva" src="https://github.com/user-attachments/assets/1194a75e-8666-4ee4-9e29-67925e69e684" />

- POST, PROPFIND, GET ja POST löytyvät täältäkin.

### g) agentti & h) pienemmät jäljet

- vaihdoin user agentin opettajan vihjettä käyttäen antamalla komennon:
```
nmap -A localhost --script-args http.useragent="Mozilla/5.0 (X11; Linux x86_64; rv:140.0) Gecko/20100101 Firefox/140.0"
```
- eli tässä tapauksessa vinkistä `http.useragent="` lähtien XBox on korvattu Mozillan tunnisteella (ks. kuva tehtävässä apache-log).
